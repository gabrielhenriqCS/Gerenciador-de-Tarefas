Atue como um Engenheiro de Software Full Stack Sênior especialista em TypeScript. Quero que você crie a base de código inicial para um gerenciador de tarefas moderno chamado "TaskFlow", integrando o Front-end e o Back-end com foco em tipagem estática fim-a-fim e validações robustas.
Siga To Do List estritamente as especificações de arquitetura e escopo abaixo:
---
### 🚀 STACK TECNOLÓGICA
- **Front-end:** React.js (Vite), TypeScript, Tailwind CSS, React Hook Form, Zod.
- **Back-end:** Node.js, Fastify, Prisma ORM, Zod.
- **Banco de Dados:** PostgreSQL.
---
### 🗄️ 1. MODELAGEM DO BANCO DE DADOS (Prisma Schema)
Gere o arquivo schema.prisma contendo:
- Enums para Priority (ALTA, MEDIA, BAIXA) e Category (TRABALHO, PESSOAL, ESTUDOS).
- Tabela User: id (uuid), email (unique), name, avatarUrl, passwordHash, createdAt.
- Tabela Task: id (uuid), title, description (opcional), status (PENDENTE/CONCLUIDA, default PENDENTE), priority, category, dueDate (DateTime), userId (FK para User), createdAt.
---
### ⚙️ 2. ESTRUTURA E CÓDIGO DO BACK-END (Fastify + Zod + Prisma)
Crie uma arquitetura modular baseada em "Features/Módulos". Quero os seguintes arquivos estruturados com TypeScript:
1. **src/config/env.ts**: Validação de variáveis de ambiente usando Zod (DATABASE_URL, PORT, JWT_SECRET).
2. **src/lib/prisma.ts**: Instância global do Prisma Client.
3. **src/modules/tasks/tasks.schemas.ts**: Schemas do Zod para validação de entrada:
   - createTaskBodySchema: title (string, min 3), description (string opcional), dueDate (string iso date), priority (enum), category (enum).
4. **src/modules/tasks/tasks.routes.ts** e **tasks.controller.ts**: Rota POST /tasks integrada com a validação do Zod, injetando os dados validados no Controller e chamando o Prisma para salvar a tarefa associada a um usuário simulado.
---
### 💻 3. ESTRUTURA DO FRONT-END (React + Vite + Tailwind)
Gere a estrutura de componentes em TypeScript baseada no seguinte layout clean e minimalista:
1. **components/layout/Sidebar.tsx**: Menu estilo barra de navegação no topo estático com links para as categorias (Trabalho, Pessoal, Estudos).
2. **components/layout/Header.tsx**: Barra superior com nome do usuário, avatar e um input de busca textual.
3. **components/common/TaskCard.tsx**: Card que exibe o título da tarefa, badge colorido baseado na prioridade (ex: vermelho para ALTA, amarelo para MEDIA), data de vencimento formatada e um Checkbox.
   - *UX de Interação:* Ao marcar o checkbox, simule o texto ficando riscado (line-through) e opaco.
4. **components/common/Modal.tsx**: Um modal flutuante de "+ Nova Tarefa" utilizando React Hook Form e Zod para validar os campos antes do envio.
5. **Estados Vazios (Empty State):** Se o array de tarefas estiver vazio, renderize um componente com uma mensagem motivacional e um design limpo centralizado na tela.
---
### 🛑 DIRETRIZES DE REQUISITOS (O que NÃO fazer)
- NÃO crie sistemas de compartilhamento de times ou colégios (o app é 100% individual).
- NÃO crie categorias dinâmicas (use apenas as três estáticas do enum).
- NÃO implemente Dark Mode nesta fase.
Forneça os trechos de código limpos, tipados, com tratamento de erro básico (try/catch) no backend enviando respostas HTTP corretas (400 para validação errada, 201 para criação).
